mysql is not required for Tracks because you can use other databases too, like sqlite. Is mysql for you a must-have? I think sqlite is better supported on arm…

all errors (tracks, rake) depend on the ruby-mysql binding that is missing. So you need to fix that first, or move away from mysql.

You can check the mkmf.log file. There is also another error log in the same directory. That should give you some clues on missing libraries for the ruby-mysql to build.

Small remark, you are running the new ruby 1.9.1. Tracks is not yet tested on ruby 1.9.1. Also the rails we’re using isn’t completely compatible with ruby 1.9, so you may run into new problems. For 1.8 we’ll upgrade rails and probably test under 1.9.
