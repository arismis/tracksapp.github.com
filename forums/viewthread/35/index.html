Hi. Are you still having this problem?  I only have a few thoughts about what it might be doing.

First, check the “Tracks” command window - the one that needs to remain open during operation.  What does it read?  When Tracks is started under U/S, it should throw lines like this:

=> Rails application started on http://0.0.0.0:3000

1) Really check ALL your browser, firewall, anti-virus protections.  My McAfee anti-virus makes me permit the .vbs script every time.  My Windows Explorer has the same reaction when U/S starts.  And make sure your firewall allows Port 3000 to be used in all permissions.

2) Try starting Uniform Server on its own before starting Tracks. To do this, use the Disk Start.vbs file.  This will start U/S and MySQL.  Then, after the Uniform Server admin pages comes up, start Tracks by using the TracksStart.bat file.

If this fails, close everything and use the Server Start.bat file to start U/S only (without MySQL).  If it gives you the U/S admin panel (it should) manually run MySQL in the left pane by clicking on “Run MySQL” .  Then run Tracks by using the TracksStart.bat file as above.

I am trying to break down the steps and see which is not taking. My bet is that it is a security/permissions issue.  Tracks is definitely running, so its not that part. My second guess is that Uniform Server is not running properly.

Last suggestion: find out your IP address (get a c:\ command prompt.  type “ipconfig’).  Substitute your IP addres for “localhost”, eg. http://192.168.1.101:3000  What. happens then?

Any other symptoms? Let me know.
