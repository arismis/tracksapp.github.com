This is definitely a bug. The repeat pattern is deleted when the context is deleted, but the todos that belong to the repeat pattern are not updated correctly. I think you have changed the context of the todo before deleting the context.

I think if you browse your context, one will fail with this error too, besides the home page, i.e. all pages on which the corrupted todo is there, will fail.

AFAICS, this can only be fixed if you manually update the todo in your database and remove the corrupted todo. Are you able to do that?
