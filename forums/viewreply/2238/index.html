no, this rewrite stuff is confusing.

The application error should leave more info in the log. If your production.log does not have more info, please look at the error_log of apache

you are now doing two things. 1) rewriting all request to 127.0.0.1 and 2) rewriting all request to dispatch.cgi

1) goes to your started server instance
2) goes to a new instance started bu dispatch.cgi

I think you should pick only one

So, backup the Tracks .htaccess and replace the current one with the original one from your webhost and see if that works.
