No, I don’t have a private server but to get shell access:

Users > Manage Users
Select ‘edit’ any user
For ‘User Account Type’ - select ‘Shell account - allows SFTP/FTP plus ssh access’
For ‘Shell Type’ I have it set to ‘\bin\bash’ though I don’t know if that matters

Can you just take a quick look at the steps I took to install and see if you see any obvious differences?

Via Terminal:

1. ssh in to my sites:
ssh .(JavaScript must be enabled to view this email address)

2. Go to the install root dir:
cd my.site.com

3. Get the latest version
git clone git://github.com/bsag/tracks.git

4. Rename database.yml.tmpl to database.yml

5. Rename environment.rb.tmpl to environment.rb

6. Edit details for database.yml:

development:
  adapter: mysql
  database: tracks
  host: localhost
  username: root
  password:

test:
  adapter: sqlite3
  database: “:memory:”

production:
  adapter: mysql
  database: mydatabase
  host: my.hostname
  username: myuser
  password: mypassword
 
7. Edit environment.rb to change:
SALT = “change-me” to SALT = “randomstring”

8. Update the database:
rake db:migrate RAILS_ENV=production

9. Here’s the main issue. I run:

script/server -e production

Well - I have it working now but I am waiting to see if it is stable. I made two changes:

1. I re-enabled Passenger in the Dreamhost panel (I originally had it enabled, then disabled it)

2. I don’t know if this was how it was supposed to be but I originally had the folders set up like:

my.domain.com/tracks/all_the_files_here

and could login and access it if I ran the webrick server at this url:

my.domain.com:3000

so I moved everything out of the tracks folder so it is like:

my.domain.com/all_the_files_here

and I can now login and access everything at my.domain.com and no longer need the ‘:3000’

I will repost if this doesn’t stay stable.
