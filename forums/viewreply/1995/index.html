Thank you Bryan for your example and for pointing me in the right direction.  I have no prior experience doing this and I was not able to get my installation working.  I ended up using a slightly different method which worked for me.

The first thing I had to do was modify my ~/.bash_profile to include my gem path by adding the line:

export GEM_PATH="$HOME/.gems:$GEM_PATH" 

I then ran a modified version of jpravetz’s script from here: http://discussion.dreamhost.com/thread-119253.html

This is what my script looked like:

#!/bin/sh

ROOT=$HOME

mkdir $ROOT/.gems
cd $ROOT

# libxml2-2.7.7 does not work, need libxml2-dev
# libxslt-1.1.26 does not work, need libxslt1-dev

# Get libxml2-dev
wget http://ftp.de.debian.org/debian/pool/main/libx/libxml2/libxml2_2.6.32.dfsg.orig.tar.gz
tar xvf libxml2_2.6.32.dfsg.orig.tar.gz

# libxslt1-dev
wget http://ftp.de.debian.org/debian/pool/main/libx/libxslt/libxslt_1.1.24.orig.tar.gz
tar xvf libxslt_1.1.24.orig.tar.gz

cd $ROOT/libxml2-2.6.32
nice ./configure --prefix=$ROOT --without-python
nice make
nice make install

cd $ROOT/libxslt-1.1.24
nice ./configure --prefix=$ROOT --with-libxml-prefix=$ROOT --without-python
nice make
nice make install

gem install nokogiri -- --with-xslt-include=$ROOT/include --with-xslt-lib=$ROOT/lib
gem install RedCloth -- --with-xslt-include=$ROOT/include --with-xslt-lib=$ROOT/lib 

Finally, I had to modify the tracks config/environment.rb file to include my gem path by adding the following line:

ENV['GEM_PATH'] = '/home/my_username/.gems:usr/lib/ruby/gems1.8/' 

I hope this might help someone else.
