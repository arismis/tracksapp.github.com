SOLVED

I had to do two things to solve this issue:

1) Change home path

Added ENV[‘HOME’] = ‘/home/username’ to my environment.rb file at the top.  This allowed the script to use the appropriate path on my web host.

After this change, I was able to send emails to my Tracks app without bounceback errors.  However, the email did not populate in my tracks database.  I did not receive any errors to indicate what the problem was.

2) Change email receive gateway to look for “To” address rather than “From”

I suspect that I may be using this functionality differently than others.  Basically, I have an email address on my webhost that I want to receive all the tasks I send to Tracks by email.  I want to be able to send them from any email address.  So, when this ‘receiving’ email on my webhost receives an incoming email, I redirect it to the ‘runner’ script.

I was redirecting email from an incoming email (i.e., .(JavaScript must be enabled to view this email address)) on my host to the script below. 

|/usr/bin/ruby /home/username/tracks/script/runner -e production MessageGateway.receive(STDIN.read)’

I had ‘tasks@webhost.com’ set as my incoming email address in Preferences. When I send an email to ‘tasks@webhost.com’, the email was redirected to ‘tasks@webhost.com’, but it showed as from the original sender. 

I figured out that the email integration functionality was looking for the sender (from:) to be the Preferences email address.  Since it was coming from the original sender, it never showed up in my task list.

So the second change I had to make was to change the field that was being compared to the Preference email address.  I changed it from “from” to “to”.  When I made that change, I was now able to receive email sent from any address to the email address ‘tasks@webhost.com’.

I don’t know if this is the right change, but it was necessary for it to work.  I’m sure that ‘epall’ will have some comments in this regard.

If you are having trouble getting the current email solution to work, I created a ruby script that checks an email address and adds todos to tracks.  Fill in the values and it should work. 
http://svn.morelightmorelight.com/trunk/personal/tracks/trackmail.rb

#!/usr/bin/ruby
# you'll probably need your own ruby gems if you are running this on a shared host.
# I did this approach: http://railstips.org/2008/11/25/rubygems-yours-mine-and-ours
require 'net/pop'
require 'rubygems'
# if you do not have this installed just type 'gem install tmail'
require 'tmail'
# important - this needs to be the path from this script
# to the tracks_api_wrapper.rb
# it is usually under YOURTRACKSINSTALL/doc/tracks_api_wrapper
require 'tracks_api_wrapper'
#mail configuration
mail_user = 'your_login@yoursite.com'
mail_password = 'y0ur_p@$$w0rd'
mail_server = 'mail.yoursite.com'
#tracks configuration
tracks_user = 'your_tracks_user'
tracks_password = 's00p3r_s3cur3_p@$$'
tracks_server = 'your.tracksinstall.com/full_path_to_it'
tracks_default_context = 3 #this is just the context we will put your mail under
Tracks::Base.site = "http://" + tracks_user +":"+ tracks_password+"@"+tracks_server
#open each mail on the server
pop = Net::POP3.new(mail_server)
pop.start(mail_user,mail_password) 
  pop.each_mail do |mail|
    #parse the mail
    email = TMail::Mail.parse(mail.pop)
    t = Tracks::Todo.new
    t.description = email.subject
    t.notes = email.body + "\nfrom: " + email.from.to_s
    t.context_id = tracks_default_context
    t.save
    #get rid of the email.  
    mail.delete
  end
pop.finish
