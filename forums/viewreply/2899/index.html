I installed Tracks on my PC running Windows 7 64. Ideally I would install it on my hosting (Hostgator), but it did not work out. (I did install Tracks 1.7.3 on Hostgator successfully)

I played around a bit with 2.0 and noted down some features/functions I would like to see in Tracks someday (only suggestions of course):
Feature/Function suggestions:

* Ideally everything should be drag’n'dropable - for lazy people, like me, for example.

** You should be able to drag action to a context, tag (cloud) or a project on the right sidebar (not only on the action list, because if I have many actions defined in all contexts, there might be a long-long way up/down to drag the action from one context to other).

** Thicker/Due feature. There could be a little calendar (current and next month) on the sidebar. One should be able to drag’n'drop an action on a specific date, or a week (a little square drop-field in front of each week in the calendar), or a month (to drop the action on the “month name” field of the calendar). Any action would then automatically hide until specific day/week/month.

** Ability to collapse/expand context/tag/project lists and calendar in case all expanded lists do not fit in the screen.

* Date stamps. When was the action created/updated (At least in the action/project “notes” section, automatically). If I want to see when the action/project was created, I can “toggle notes” for all or particular action/project to find this information.

* Missing tickler buttons +1 and +7 on the right. It was a great feature.

* Context groups in Project view. To see specific context in each project. There could also be a context picture/icon on the left/right of each action (like a little home, computer, phone etc.)

* One should be able to hide/unhide a context by clicking on the Visible/Hidden button on the right. Similarly in projects one could click the Active/Hidden/Completed button to change the status without going in the edit section.

* Sort by tags alphabetically or by number of actions per tag. Right now we can display action from one tag by clicking on a tag in the action. I would also appreciate a tag cloud in the sidebar.

* Make project groups. We all have many small projects (multi-action tasks) that are a part of a bigger project. It would be nice if we could see the BIG picture with all the small projects and actions to complete this project. Also I should be able to make dependencies for projects also, so that depending project would be hidden until specific projects have been completed.

* Add files option. Add files to an action or project notes. Even more sophisticated if we could link to AmazonS3 bucket and store all files there. (*Update* There seems to be a simple solution to do that: Amazon S3 Uploader with Paperclip)

* Left sidebar should be sticky - stay in place even if I scroll trough my actions. This feature is only relevant if I can drag’n'drop actions on contexts, tags, dates. Also in this case there should be the ability to collapse/expand context/tag/project lists and calendar in case all expanded lists do not fit in the screen.

* Inbox section. A dedicated section for adding actions/notes without context. Made to get actions/tasks in system as soon, as simple and as easy as possible. For example - a shortcut key (alt+n or something) to start new record(action). First line should be automatically converted to a title but next lines to a description of a note/action. When I have all my actions in INBOX section I can start organizing them by project, context, due, someday/maybe, calendar, waiting, next action or a reference (according to GTD). The simplest way to organize would of course be dragging and dropping on a context, project, calendar, tag etc.

* Context suggestion in project properties “default context” field.

* Tag suggestions in project properties “default tags” field.

There were also some issues:
* If an action doesn’t have a context, it cannot be dragged and dropped. I know, When creating new action you can’t leave context blank, but If I mark an action as done and then unmark it from completed actions list, this action looses context and then it cannot be dragged and dropped until I specify a context again.

* Multiple next actions. I tried to add more actions, but the limit is only 100 characters - not very usefull. You should increase it at leas to 1000

* Issues with installation. Issues covered in posts :http://www.getontracks.org/forums/viewthread/840/ and http://www.getontracks.org/forums/viewthread/887/ I man.aged to install v2.0RC2 on windows 7 64 with Ruby installer 1.8.7. Ruby 1.9.2 did not work for me.


With features above and probably some additional ones, Tracks might be the ULTIMATE GTD system. I’m looking forward to it!
My Kudos to developers!

I was able to set up 2.0RC2 with very little trouble on an Ubuntu 10.04 (LTS) server. The only non-obvious requirement (as suggested by someone else on page 1 of this thread) was I had to ‘apt-get install libxslt-dev’ before I could install sanitize.

As a preface to the following: I will upload the bugs below to assembla, trying to sort by appropriate components. Listing them here in case I get hit by a bus tomorrow before I can add them.

—-

I am quite impressed at the improvements.

I’m looking forward to trying the google gmail gadget more fully.

I tried this on firefox (3.6.16), Safari (5.0.5), and iPhone safari (whatever the latest update was). A few things below seem to be problems only in firefox (labeled as “FF”).

I did not see the bug reported by someone on this thread concerning losing characters at the beginning of an entry.

Calendar: has hardcoded German text? or is there some configuration I haven’t found?

I really like how dependencies work, including the “view successors” arrow.

completed items: unlike 1.7, newly completed items appear at the bottom of the “completed” list instead of the top, out of sort order.

Clicking on the pencil, x, or star seems to overlay the spinner in a way that looks like there are display artifacts: it might be cleaner to not have transparency to the pencil/x/star underneith? Alternately, I sort of miss the throbbing pencil from 1.7

edit an item, clicking “Update” or in new item, clicking “Add Action” cause an oddly shaped dark-grey background to appear (only in FF). Versus edit an item, “Cancel” which causes a properly shaped background.

Add multiple next actions: the last instruction repeats: “Tags for all actions (sep. with commas) (separate with commas)”

new item, “depends on”: if you fill in something that doesn’t exist, it silently ignores it. I would prefer if it gave me a warning and didn’t let me save.

The dropdown system is excellent; much better than 1.7’s. there is one difference between “depends on” and Project/Context that feels like a bug: “depends on” will not default to the first value in the list while I’m typing. So I have to hit down, then tab, rather than just tab.

I appreciate how much better the new dropdowns work on my iphone.

(only in FF) Editing an item: if you Add another dependency, the new “Add another dependency” box has the spinner on the right-hand side whether or not you’ve typed any text. I also got it to show the spinner for the “Depends on” on the right-side, though I wasn’t able to reproduce that just now.
