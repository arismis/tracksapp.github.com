Didn’t think your title was mean at all. Just pointing out that if you say it is idiot proof, nature will produce a better idiot to prove you wrong.

Everything I had to do in order to make it work was in the Tracks install manual. So it is not a failing in your PDF at all. Just some stuff you might want to mention.

First was the database.yml. You had to remove the tmpl extension and set up the SQLite DB (of which Tracks does come with). So mine looks like this now:

test:
    adapter: sqlite3
    database: ":memory:"

production:
  adapter: sqlite3
  database: db/tracks.db 

Also I needed to change the “site.yml.tmpl” file by removing the extension and setting up two variables. First was the “Salt” and the second was the time-zone. Ruby kept complaining about the time zone until I figured out what I was supposed to be putting in there.

Again…all figured out after reading the tracks manual. The last time I used Tracks I had installed it on a OS X server, so I should have remembered both of these things, but that was quite a while ago.
